OVERVIEW

After importing the package, you will have an additional MenuItem "Custom Script Templates" under the "Create"-Menu.

(When you right-click inside the Project window)

This MenuItem holds all Templates.

You can customize the default ones, delete items you don't want, and even create your own Templates.

TEMPLATES

Templates are .txt Files inside the Templates Folder

Custom Script Templates -> Settings -> Templates

The most important tag is #SCRIPTNAME#.

This is the placeholder for the name you give the File.

The #NAMESPACE#-tag is optional.

When you put it in the Template, it will insert your Projects root Namespace at that line,

Edit -> Project Settings -> Editor -> Root namespace

or the Namespace of the next Assembly Definition File in your Project.

(Aseembly Definition Reference also works)

If your Project or the Assembly Definition File doesn't have a root Namespace set, the tag will be removed on File creation.

So you can just leave the Tag in the Template, it won't break anything.

You can also set a custom Namespace, regardless if you have a root Namespace set.

#NAMESPACE#CustomNamespace#

Just write the name of the Namespace after the #NAMESPACE#-tag and end the line with a #

Both tags, #SCRIPTNAME# and #NAMESPACE# have to be upper case and be in between 2 #

You can write anything inside a Template, from using statements to Methods, everything works!

SETTINGS

You can find the "Settings"-ScriptableObject under:

Custom Script Templates -> Settings

The Namespace settings are pretty self explanatory, just hover over them with your mouse, to get a tooltip with what they do.

The Menu Item settings are for the display name in the "Create"-Menu and its position.

The Templates list contains all .txt Templates you want to see as a Menu Item.

So after you create a new .txt Template, add it to this List and hit the "Compile Menu Items"-Button.

You only have to press the "Compile Menu Items"-Button when you made changes in the "Menu Item Settings" or the "Templates List".

You don't have to press it after making changes in the "Namespace Settings".

IMPORTANT

In future updates, don't import the "Settings"-Folder!

Your Templates and personalized Settings will be overwritten with the default ones if you also import this Folder.

Just uncheck the "Settings"-Folder in the Import settings.
